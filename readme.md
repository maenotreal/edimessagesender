# EDI Message Sender

![GitHub release](https://img.shields.io/github/v/release/maenotreal/edimessagesender)

Консольный инструмент для работы с [Kontur.EDI API](https://kontur.ru/edi). Позволяет генерировать и отправлять EDI-документы (ORDERS, RECADV, PRICAT), работать с каталогами, просматривать входящие сообщения и автоматически отвечать на PORDERS / DESADV в режиме прослушивания.

> **Дисклеймер:** Это внутренний инструмент, разработанный для личного использования. Он не предназначен для внешних пользователей и не предоставляет никаких гарантий стабильности, безопасности или поддержки.
>
> Проект создан в процессе vibe-кодинга — большая часть кода написана совместно с AI-ассистентом. Используйте на свой страх и риск.

## Возможности

| Режим | Описание |
| ----- | -------- |
| **1** | Сгенерировать ORDERS из конфига и отправить |
| **2** | Отправить существующий XML-файл |
| **3** | Режим прослушивания — авто-ответ на PORDERS и DESADV |
| **4** | Работа с каталогами (TradeItemTableLayout / PRICAT) |
| **5** | RECADV — уведомление о приёмке товара |
| **6** | Хранилище — просмотр ORDERS / DESADV |

---

## Режимы

### 1 — ORDERS из конфига

Генерирует ORDERS на основе `edi_config.json` (GLN покупателя, продавца, товарные позиции) и отправляет через API. Ответ открывается в мониторинге Kontur.EDI, ORDERS сохраняется в хранилище.

### 2 — Отправить существующий XML

Открывает диалог выбора файла. Отправляет любой готовый XML-документ через API. Если файл содержит ORDERS — предлагает сохранить в хранилище.

### 3 — Режим прослушивания

Бесконечный цикл опроса входящих сообщений (каждые 30 сек):

- **PORDERS → ORDERS** — при получении PORDERS автоматически формирует и отправляет ORDERS. Сохраняет сценарий обработки RECADV из поля `seller/additionalIdentificator`.
- **DESADV → RECADV** — при получении DESADV автоматически формирует и отправляет RECADV. Количества рассчитываются по сценарию, сохранённому при обработке соответствующего PORDERS.

**Сценарии RECADV** (`additionalIdentificator` в PORDERS):

| Значение | Поведение |
| -------- | --------- |
| `NOCHANGE` | `acceptedQuantity = despatchedQuantity` (по умолчанию) |
| `REJECT` | `acceptedQuantity = 0` — все позиции отклонены |
| `ADD_QTY` | `acceptedQuantity = despatchedQuantity × 1.2` (+20%) |

Состояние (последний обработанный event_id) сохраняется в `listener_state.json`. Выход — `Ctrl+C`.

**Логи слушателя:**

| Файл | Содержимое |
| ---- | ---------- |
| `logs/log_YYYY-MM-DD.txt` | Всё: HTTP-трафик, опросы, события |
| `logs/listener_YYYY-MM-DD.txt` | Только события слушателя (без HTTP-шума) |

### 4 — Каталоги PRICAT

Загружает Excel-файл с позициями товаров и генерирует XML каталога формата TradeItemTableLayout. Поддерживает два режима:

- **Добавление** — отправка `01.xml` → `03.xml`
- **Удаление** — отправка `03.xml` (status=Deleted) → `01.xml`

### 5 — RECADV

Интерактивный режим формирования RECADV на основе ранее полученного DESADV. Позволяет выбрать ORDERS из хранилища, обновить его статус, загрузить новые DESADV из ящика и скорректировать принятые количества по каждой позиции вручную перед отправкой.

### 6 — Хранилище

Локальная база отправленных ORDERS и привязанных к ним DESADV. Возможности:

- Просмотр статусов ORDERS (ORDRSP: Accepted / Rejected / Changed)
- Обновление статуса и загрузка новых DESADV из ящика
- Удаление отдельных записей
- Очистка завершённых (все RECADV отправлены) или устаревших ORDERS

## Быстрый старт

```bash
# Первичная настройка
python setup.py

# Запуск
python main.py
```

---

## Конфигурация

Файл `edi_config.json` создаётся автоматически через `setup.py`.

Поддерживаются два режима авторизации:

### OIDC (рекомендуется)

```json
{
  "auth_mode": "oidc",
  "oidc": {
    "client_id": "...",
    "client_secret": "..."
  }
}
```

### Legacy

```json
{
  "auth_mode": "legacy",
  "legacy": {
    "api_client_id": "...",
    "login": "...",
    "password": "..."
  }
}
```

---

## Обновление

Приложение проверяет обновления автоматически при запуске, либо вручную через меню (`u`).
Новая версия скачивается с GitHub Releases и применяется поверх текущей папки.

## Структура проекта

```text
main.py           — главное меню и обработчики режимов
config.py         — загрузка и валидация конфигурации
auth.py           — авторизация (OIDC / Legacy)
api.py            — клиент Kontur.EDI API
xml_builder.py    — генерация ORDERS / PRICAT XML, разбор PORDERS
recadv_builder.py — генерация RECADV XML, сценарии обработки
store.py          — локальное хранилище документов (edi_store.json)
updater.py        — автообновление
setup.py          — мастер первичной настройки
logger.py         — настройка логирования (основной + listener лог)
tests.py          — юнит-тесты (pytest)
```
