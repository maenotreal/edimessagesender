# EDI Message Sender

Консольный инструмент для работы с [Kontur.EDI API](https://kontur.ru/edi). Позволяет генерировать и отправлять EDI-документы (ORDERS, RECADV), работать с каталогами и просматривать входящие сообщения.

---

## Возможности

| Режим | Описание |
| ----- | -------- |
| **1** | Сгенерировать ORDERS из конфига и отправить |
| **2** | Отправить существующий XML-файл |
| **3** | Тестовый ORDERS с произвольными GLN |
| **4** | Работа с каталогами (TradeItemTableLayout) |
| **5** | RECADV — уведомление о приёмке товара |
| **6** | Хранилище — просмотр ORDERS / DESADV |

---

## Требования

- Python 3.10+
- Зависимости устанавливаются автоматически при первом запуске

---

## Быстрый старт

```bash
# Первичная настройка
python setup.py

# Запуск
python main.py
```

---

## Конфигурация

Файл `edi_config.json` создаётся автоматически через `setup.py`.

Поддерживаются два режима авторизации:

### OIDC (рекомендуется)

```json
{
  "auth_mode": "oidc",
  "oidc": {
    "client_id": "...",
    "client_secret": "..."
  }
}
```

### Legacy

```json
{
  "auth_mode": "legacy",
  "legacy": {
    "api_client_id": "...",
    "login": "...",
    "password": "..."
  }
}
```

---

## Обновление

Приложение проверяет обновления автоматически при запуске, либо вручную через меню (`u`).
Новая версия скачивается с GitHub Releases и применяется поверх текущей папки.

---

## Структура проекта

```text
main.py           — главное меню и обработчики режимов
config.py         — загрузка и валидация конфигурации
auth.py           — авторизация (OIDC / Legacy)
api.py            — клиент Kontur.EDI API
xml_builder.py    — генерация ORDERS XML
recadv_builder.py — генерация RECADV XML
store.py          — локальное хранилище документов
updater.py        — автообновление
setup.py          — мастер первичной настройки
```
